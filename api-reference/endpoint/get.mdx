---
title: "Get User"
api: "GET https://api.linkvite.io/v1/user"
description: "This endpoint gets a user."
---

### Response

<ResponseField name="ok" type="boolean">
  Indicates whether the call was successful. true indicates success, false indicates failure.
</ResponseField>

<ResponseField name="message" type="string">
  A message indicating the result of the call.
</ResponseField>

<ResponseField name="data" type="object (optional)">

The user object, if the call was successful. If the call was unsuccessful, this field will not be present.

<Expandable title="Toggle object">

<ResponseField name="id" type="string">
  This is the user id (user_id or USER_ID) that will be used to identify which user is viewing the dashboard.
</ResponseField>

<ResponseField name="name" type="string">
    This is the user's name.
</ResponseField>

<ResponseField name="email" type="string">
    This is the user's email.
</ResponseField>

<ResponseField name="username" type="string">
    This is the user's username.
</ResponseField>

<ResponseField name="avatar" type="string">
    This is a URL that points to the user's avatar.
</ResponseField>

<ResponseField name="emailVerified" type="boolean">
    True indicates that the user's email has been verified, false indicates that the user's email has not been verified.
</ResponseField>

<ResponseField name="accountType" type="string">
    This is the user's account type. The account type can be free, pro, or enterprise.
</ResponseField>

<ResponseField name="status" type="string">
    This is the user's status. The status can be active, inactive, or suspended.
</ResponseField>

<ResponseField name="verified" type="boolean">
    This is an internal field that indicates whether the user has been verified. You don't need to worry about this field.
</ResponseField>

<ResponseField name="folderName" type="string">
    This is the user's folder name. It's the page title for /folder, and can be edited by the user.
</ResponseField>

<ResponseField name="privateAccount" type="boolean">
    True indicates that the user's account is private, false indicates that the user's account is public.
</ResponseField>

<ResponseField name="createdAt" type="string">
    This is the date and time that the user was created. It's in ISO 8601 format. (YYYY-MM-DDTHH:mm:ss.sssZ)
</ResponseField>

<ResponseField name="updatedAt" type="string">
    This is the date and time that the user was last updated. It's in ISO 8601 format. (YYYY-MM-DDTHH:mm:ss.sssZ)
</ResponseField>

<ResponseField name="lastLogin" type="string">
    This is the date and time that the user last logged in. It's in ISO 8601 format. (YYYY-MM-DDTHH:mm:ss.sssZ)
</ResponseField>

</Expandable>

</ResponseField>
